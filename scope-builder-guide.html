01 — Overview

What this tool does

The Project Scope Builder helps you create professional, branded project scopes that can be pushed directly into Salesbuildr quotes. It handles the full workflow — from loading a preset or describing a project in plain English, through building and refining the task list, to generating a polished HTML widget ready for your customer-facing proposal.

🗂 Three-panel layout

Left panel — all your controls: presets, projects, team settings, brand color, and publishing actions.

Center panel — your working area: project overview, task grid, and exclusions. This is where you build the scope.

Right panel — the AI assistant. Describe a project and it generates a full scope, or ask it to review what you've already built.

🎨 Brand color

Set your brand color once in the left panel — it applies to every widget you generate. Your table headers, section headings, and accent elements will always match your Salesbuildr branding. It saves automatically so you never need to set it again.

02 — Storage

Local vs Team mode

The tool works in two modes depending on whether you enter a team passphrase. Understanding the difference is important before you start saving templates.

💾 Local
Solo use

No passphrase needed. Templates are saved to your browser on your device. Only you can see them, and only on the machine you saved them on.

🔗 Team
Team use

Enter a shared passphrase. Templates are saved to a shared server store. Anyone on your team who enters the same passphrase sees the same templates — on any device, any browser.

Feature 💾 Local (no passphrase) 🔗 Team (with passphrase)
Works without setup ✓ Yes Requires passphrase
Templates visible to colleagues ✗ No ✓ Yes
Works on any device ✗ This browser only ✓ Any device
Templates survive clearing browser data ✗ No ✓ Yes
Export / Import JSON ✓ Always available ✓ Always available
💡
Which should I use? If you work alone or are just exploring the tool, Local mode is fine. If two or more people in your organisation will be creating scopes and sharing templates, set up a team passphrase so everyone works from the same library.
03 — Getting Started

First-time setup

Takes about two minutes. Do this once and the tool remembers your preferences every time.

1
Set your brand color

In the left panel under Brand color, click one of the preset swatches or enter a custom hex code to match your Salesbuildr branding. This is saved automatically and applies to every widget you generate.

2
Enter your team passphrase (team use only)

If you work with colleagues, find the Team passphrase field in the left panel under Team. Enter the passphrase your organisation has agreed on. The badge will change from 💾 Local to 🔗 Team confirming you are connected to the shared template store.

3
Save your Salesbuildr credentials (optional)

Click Save to Salesbuildr in the left panel, enter your API Key and Integration Key, and tick Remember credentials. Your keys are saved to this browser so you only need to do this once per device.

4
Load a preset or start from scratch

Under Load a preset, choose a project type that matches your engagement and click the arrow. The tool loads a full task list, overview, and exclusions as a starting point. You can also start with a blank scope using New project and let the AI build it from a description.

⚠️
Important — team passphrase: You need to re-enter your passphrase each time you open the tool. It is not stored between sessions by design. Keep your passphrase somewhere your team can access it — a shared password manager or internal wiki works well. If it is lost, your templates are not deleted but will be inaccessible until the passphrase is recovered.
04 — Templates

Saving and loading templates

Templates let you save a complete scope — tasks, overview, exclusions, and settings — and reload it as a starting point for future engagements of the same type. They are different from presets: presets are built in, templates are ones you create and own.

1
Save a template

Build or refine a scope until it reflects your standard approach for that project type. Click Save as template in the left panel, give it a name, and confirm. If you are in Team mode, it is immediately available to all colleagues using the same passphrase.

2
Load a template

Select a template from the Templates dropdown and click the arrow. The current scope will be replaced with the saved template. You will be asked to confirm before anything is overwritten.

3
Delete a template

Select the template from the dropdown and click the 🗑 button. In Team mode, deleting a template removes it for everyone — you will be asked to confirm before this happens.

✅
Best practice: Treat templates as your refined, quality-controlled starting points — not rough drafts. Load a preset, adjust it to match how your team actually delivers that project type, then save it as a template. Future scopes for that project type start from your standard, not the generic preset.
📦 Export and Import JSON

Every scope can be exported as a JSON file using Export JSON in the left panel. This is useful for sharing individual scopes outside the team template system, keeping backups, or transferring work between organisations. Use Import JSON to load a previously exported file.

05 — Projects

Managing current projects

The Current projects section in the left panel shows every scope you are actively working on. You can switch between them without losing your work — the tool auto-saves before switching.

🔄 Switching between projects

Click any project in the list to switch to it. Your current project is saved automatically before the switch happens — you will never lose work by switching projects. The active project is highlighted with a blue border.

➕ Starting a new project

Click + New project at the bottom of the projects list. The current project is saved first, then a blank scope opens. Give it a title and customer name in the left panel to help identify it in the list.

🏷 Shared vs local projects

Each project in the list shows a shared or local badge. A project becomes shared when it is saved while a team passphrase is active. Local projects are visible only on the current device.

06 — AI Assistant

Using the AI assistant

The AI assistant lives in the right panel and can generate a complete scope from a plain-English description or review and improve an existing one.

✨
Generate scope from description

Click the Generate scope from description chip, then describe the project in plain English — for example: "Migrate a 3-server office to Azure, small team, 6-week deadline." The AI generates a full task list, project overview, and exclusions. Review the preview and click Apply to scope to load it into the editor.

🔍
Review my current scope

Click Review my current scope to ask the AI to analyse what you have already built. It returns specific feedback — flagging tasks that may be under-estimated, missing items that are typically needed, and anything that could cause scope creep. You can add its suggested tasks directly to the grid with one click.

⏱
Adjust for team size or deadline

Click Adjust for team / deadline and describe your constraints — for example: "We only have two engineers and need to complete this in four weeks." The AI reviews your current scope against those constraints and suggests adjustments.

💡
Tip: The AI generates a solid starting point but your expertise matters. Always review AI-generated tasks before presenting to a customer — adjust hours to reflect your team's actual delivery pace and remove anything that does not apply to this specific engagement.
07 — Output

Generating output and pushing to Salesbuildr

When your scope is ready, click Generate scope widget in the left panel. The HTML output and a live preview appear below the task grid.

1
Choose what to show

Before generating, use the Show in customer output toggles to control whether Role, Hours, and Notes appear in the output. Role and Notes are on by default. Hours are off by default — turn them on if your customer needs to see the effort breakdown.

2
Generate and review the preview

Click Generate scope widget. Check the Preview panel on the right to see how it will look in Salesbuildr. The preview uses your brand color across headers, section titles, and the totals row.

3
Push to Salesbuildr

Click Save to Salesbuildr in the left panel, enter an optional label, and click Push →. The widget is saved directly to your Salesbuildr account and is immediately available to add to a quote. Alternatively, click Copy HTML and paste it manually via the Source Code button in TinyMCE.

08 — FAQ

Common questions

I saved templates before but can't see them now.
Check whether you are in Local or Team mode. If you saved templates without a passphrase they are stored locally in your browser — they will not appear when a passphrase is entered. If you saved them in Team mode, make sure you have entered the exact same passphrase used when saving.
A colleague saved a template but I can't see it.
Make sure you are both entering the same passphrase exactly — it is case-sensitive. Once you enter the correct passphrase the badge changes to 🔗 Team and the shared templates appear in the dropdown.
What happens if we forget our team passphrase?
Your templates are not deleted — they remain on the server. But they are inaccessible without the passphrase. Keep it in a shared password manager or internal wiki. If the passphrase is genuinely lost, the recovery path is to export any templates individual team members still have locally as JSON files, agree on a new passphrase, and re-import them.
Does the passphrase need to be secure?
It does not need to be complex — it is not protecting sensitive financial or personal data. Something memorable and unique to your organisation is ideal. Think of it like a shared Wi-Fi password rather than a banking password.
Can two different organisations use the tool without seeing each other's templates?
Yes. Each organisation's templates are completely isolated by their passphrase. There is no way for one organisation to see another's templates unless they know the exact passphrase.
The preview looks slightly different to what appears in Salesbuildr.
The preview is approximate — Salesbuildr applies its own base typography and may render fonts slightly differently. The structure, colors, and content will be correct. For the most accurate view, push to Salesbuildr and preview it in a quote.
Can I use the tool on my phone or tablet?
The tool is optimised for desktop use — the three-column layout needs a reasonably wide screen to be comfortable. It will work on a tablet in landscape mode but is not recommended for phones.
Is my data backed up?
Team templates are stored server-side and are not affected by browser changes. Local templates and projects are in your browser's localStorage — they can be lost if you clear your browser data. Use Export JSON regularly to keep backups of important scopes.
